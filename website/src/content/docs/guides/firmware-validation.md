This guide details the internal tools used to validate the ELRS Mobile application's firmware assembly logic against official "Golden" binaries.

## Ground Truth vs. Configurator Binaries

When running validation audits, you may notice that binaries generated by ELRS Mobile perfectly match the official **ExpressLRS Web Flasher**, but diverge slightly from binaries built using the local **ExpressLRS Configurator (Python)**.

This is expected behavior. The local Configurator uses Python's `json.dumps()` to assemble configuration metadata, which injects whitespace after colons and commas (e.g., `{"uid":, "domain": 1}`). ELRS Mobile and the official Web Flasher use dense, minified JSON (e.g., `{"uid":,"domain":1}`). 

The ESP32's internal C++ JSON parser ignores whitespace, meaning both binaries execute identically on hardware despite failing a strict byte-for-byte SHA256 comparison.

## 1. ExpressLRS Logic Validator (Dart)

**Location:** `firmware_testing/scripts/dart/logic_validator.dart`

The Logic Validator is a high-performance bit-level comparison tool. It is designed to verify that ELRS Mobile's internal patching and assembly logic (implemented in `package:binary`) perfectly mirrors the official ExpressLRS logic.

### Key Features
- **Bit-Level Analysis**: Pinpoints exact bit-offset failures within discrepant bytes.
- **Interactive Selection**: Lists all binaries found in `firmware_testing/binaries/` for easy comparison.
- **Zero-Copy Memory Management**: Optimized for high-speed analysis of large firmware files.

### How to Use
1. **Prepare Binaries**: Place the generated `.bin` or `.gz` files in `firmware_testing/binaries/`.
2. **Setup**:
   ```bash
   cd firmware_testing/scripts/dart/
   dart pub get
   ```
3. **Run**:
   ```bash
   dart run logic_validator.dart
   ```
4. **Follow Prompts**: Select the two files you wish to compare by their numeric index.

### Interpreting Results
Discrepancies are logged in `firmware_testing/logs/` with a report containing:
- Hexadecimal offset of the mismatch.
- Hex and Binary representation of the byte in both files.
- Specific Bit index that triggered the failure.

## 2. Ground Truth Auditor (Python)

**Location:** `firmware_testing/scripts/python/audit_tool.py`

The Auditor is a standalone, zero-dependency script intended for "Ground Truth" verification. It parses the 2704-byte configuration block appended to the end of every ExpressLRS firmware image.

### Purpose
It verifies that the metadata actually flashed to the device (Product Name, Lua Name, Hardware Layout, and Options) is correct, regardless of any layout differences in the base firmware code.

### How to Use
1. **Run directly**:
   ```bash
   python3 firmware_testing/scripts/python/audit_tool.py [file1] [file2]
   ```
2. **Interactive Mode**: If no files are supplied, the script mirrors the Dart validator by listing available binaries in `firmware_testing/binaries/`.

### Auditing Logic
1. **Integrity Check**: Performs an initial SHA-256 comparison. If hashes match, no deeper audit is required.
2. **Block Dissection**: If hashes differ, it extracts fields like `Hardware JSON` and `Options JSON`.
3. **Smart Comparison**: It automatically ignores the randomized `flash-discriminator` field within the JSON metadata, which changes every time a firmware is built.

Any actual divergences in hardware configuration or user options are reported to the console and saved to `firmware_testing/logs/`.

## 3. Binary Cross-Validation Workflow

To ensure ELRS Mobile is assembling firmware correctly, developers should periodically perform a cross-validation audit against a "Ground Truth" binary compiled on a PC.

### Step 1: Export Binary from ELRS Mobile
1. Open the **ELRS Mobile** app on your device.
2. Navigate to **Settings** > **Advanced**.
3. Toggle **Expert Mode** to **ON**.
4. Return to the **Flasher** tab.
5. Select your desired **Target**, **Version**, and configure your **Options** (Binding Phrase, WiFi SSID, and WiFi Password).
6. Click the **DOWNLOAD BINARY** button (this button only appears when Expert Mode is enabled).
7. Use the system file picker to save the `.bin` (or `.gz`) file and transfer it to your PC.

### Step 2: Generate PC Ground Truth
1. Open the [ExpressLRS Configurator](https://github.com/ExpressLRS/ExpressLRS-Configurator) on your PC.
2. Select the **exact same** Target and Version used in the app.
3. Configure the **exact same** Binding Phrase and WiFi credentials.
4. Click **Build** and save the resulting firmware file.

### Step 3: Run the Audit
1. Place both the app-exported binary and the PC-compiled binary into the `firmware_testing/binaries/` folder of this repository.
2. Run the **Ground Truth Auditor**:
   ```bash
   python3 firmware_testing/scripts/python/audit_tool.py
   ```
3. Select the two binaries from the list. 
4. Verify that the **Hardware JSON** and **Options JSON** match perfectly.
